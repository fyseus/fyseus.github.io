LeetCode第 50 场双周赛

Created2022-01-21|Updated2022-01-21
|Post View:

不知道说什么

小想法

好想出去找个实习，但是春节要到了，毕设还没怎么弄，并且感觉自己的实力还是不太够。。

总体情况

做得很开心，之前在某次每日一题中学到的乘法逆元在这次最后一题中用到了。。

题目

最少操作使数组递增（easy）

给你一个整数数组 nums （下标从 0 开始）。每一次操作中，你可以选择数组中一个元素，并将它增加 1 。

  • 比方说，如果 nums = [1,2,3] ，你可以选择增加 nums[1] 得到 nums = [1,**3**,3] 。

请你返回使 nums 严格递增 的 最少 操作次数。

我们称数组 nums 是 严格递增的 ，当它满足对于所有的 0 <= i < nums.length - 1 都有 nums[i] < nums[i+1] 。一个长度为 1 的数组是严格递增的一种特殊情况。

遍历，然后根据数组当前和上一个值大小求差。。

统计一个圆中点的数目（medium）

  • 给你一个数组 points ，其中 points[i] = [xi, yi] ，表示第 i 个点在二维平面上的坐标。多个点可能会有 相同 的坐标。

    同时给你一个数组 queries ，其中 queries[j] = [xj, yj, rj] ，表示一个圆心在 (xj, yj) 且半径为 rj 的圆。

    对于每一个查询 queries[j] ，计算在第 j 个圆 内 点的数目。如果一个点在圆的 边界上 ，我们同样认为它在圆 内 。

    请你返回一个数组 answer ，其中 answer[j]是第 j 个查询的答案。

很简单，直接对于每一个圆，遍历每一个点，求是否在圆内。

每个查询的最大异或值（medium）

给你一个 有序 数组 nums ，它由 n 个非负整数组成，同时给你一个整数 maximumBit 。你需要执行以下查询 n 次：

  1. 找到一个非负整数 k < 2maximumBit ，使得 nums[0] XOR nums[1] XOR ... XOR nums[nums.length-1] XOR k 的结果 最大化 。k 是第 i 个查询的答案。
  2. 从当前数组 nums 删除 最后 一个元素。

请你返回一个数组 answer ，其中 answer[i]是第 i 个查询的结果。

也挺简单的，求最大，就是对于后maximumBit位，异或后结果为全1，那么直接与这个全1异或就行了。

使字符串有序的最少操作次数（hard）

给你一个字符串 s （下标从 0 开始）。你需要对 s 执行以下操作直到它变为一个有序字符串：

  1. 找到 最大下标 i ，使得 1 <= i < s.length 且 s[i] < s[i - 1] 。
  2. 找到 最大下标 j ，使得 i <= j < s.length 且对于所有在闭区间 [i, j] 之间的 k 都有 s[k] < s[i - 1] 。
  3. 交换下标为 i - 1 和 j 处的两个字符。
  4. 将下标 i 开始的字符串后缀反转。

请你返回将字符串变成有序的最少操作次数。由于答案可能会很大，请返回它对 109 + 7 取余 的结果。

这一题有点意思，用到了乘法逆元，同时它那个描述就是31. 下一个排列的解法，其实就是C++中的next_permutation反过来，姑且称为previous_permutation。

然后求乘法逆元，过程中用到了费马小定理，可以很快得到数值a的乘法逆元就是pow(a, MOD - 2)。

过程中求总数，用到了排列组合的思想，从后往前排列，对于某一位，其后面都是已经排列好的，那么就能计算出总的在此之前的排列数有几种，然后算出来加到ans中，最后得到结果。

小情绪

有点孤零零的赶脚。
