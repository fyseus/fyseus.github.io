前言

因为最近放寒假了嘛，所以我想一天打两场模拟周赛也不是不可以，所以就上了。

本次模拟周赛少见的AC了，可喜可贺，可喜可贺！

题目

找出所有子集的异或总和再求和（easy）

一个数组的 异或总和 定义为数组中所有元素按位 XOR 的结果；如果数组为 空 ，则异或总和为 0 。

  • 例如，数组 [2,5,6] 的 异或总和 为 2 XOR 5 XOR 6 = 1 。
  • 给你一个数组 nums ，请你求出 nums 中每个 子集 的 异或总和 ，计算并返回这些值相加之 和 。

注意：在本题中，元素 相同 的不同子集应 多次 计数。

数组 a 是数组 b 的一个 子集 的前提条件是：从 b 删除几个（也可能不删除）元素能够得到 a 。

easy题且nums.length <= 12，直接状态压缩+枚举解决

构成交替字符串需要的最小交换次数（medium）

给你一个二进制字符串 s ，现需要将其转化为一个 交替字符串 。请你计算并返回转化所需的 最小 字符交换次数，如果无法完成转化，返回 -1 。

  • 交替字符串 是指：相邻字符之间不存在相等情况的字符串。例如，字符串 “010” 和 “1010” 属于交替字符串，但 “0100” 不是。

任意两个字符都可以进行交换，不必相邻 。

分类讨论，统计输出最小不匹配值/2;

找出和为指定值的下标对（medium）

给你两个整数数组 nums1 和 nums2 ，请你实现一个支持下述两类查询的数据结构：

  1. 累加 ，将一个正整数加到 nums2 中指定下标对应元素上。
  2. 计数 ，统计满足 nums1[i] + nums2[j] 等于指定值的下标对 (i, j) 数目（0 <= i < nums1.length 且 0 <= j < nums2.length）。
    实现 FindSumPairs 类：
  • FindSumPairs(int[] nums1, int[] nums2) 使用整数数组 nums1 和 nums2 初始化 FindSumPairs 对象。
  • void add(int index, int val) 将 val 加到 nums2[index] 上，即，执行 nums2[index] += val 。
  • int count(int tot) 返回满足 nums1[i] + nums2[j] == tot 的下标对 (i, j) 数目。

根据其数据量分析，可以发现nums1数组的长度很短，而nums2数组的长度却很长，由此可以存下nums2到一个map中，每次查询时遍历nums1，到map中找有无相应的nums2的值符合要求，增加的话，改map中相应的数量（T了一发/W了一发，才AC，┭┮﹏┭┮）；

恰有 K 根木棍可以看到的排列数目（hard）

有 n 根长度互不相同的木棍，长度为从 1 到 n 的整数。请你将这些木棍排成一排，并满足从左侧 可以看到 恰好 k 根木棍。从左侧 可以看到 木棍的前提是这个木棍的 左侧 不存在比它 更长的 木棍。

  • 例如，如果木棍排列为 [1,3,2,5,4] ，那么从左侧可以看到的就是长度分别为 1、3 、5 的木棍。

给你 n 和 k ，返回符合题目要求的排列 数目 。由于答案可能很大，请返回对 109 + 7 取余 的结果。
